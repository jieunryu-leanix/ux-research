ADR Feature Gap: LeanIX vs. Ardoq

Target Architecture Planning — UX Comparison

Priority 3 — Decision ↔ Roadmap Link
🔗
Why this matters for Architecture Decisions UX A decision without a roadmap context is orphaned documentation. The UX question is: how does an accepted Architecture Decision become a visible milestone or constraint on the target state? LeanIX and Ardoq answer this very differently.
Dimension SAP LeanIX Ardoq
Scenario branching BTM add-on (separate module) ⚠️ Native "Git-style" branch — no extra cost ✅
Decision → roadmap link Via Initiative Fact Sheet (manual link) ADR linked to initiative → linked to Jira → delivery status visible
Diff / comparison view Diagram embedding (late 2025, maturing) Side-by-side scenario diff, visual change overlay
Crowdsourcing input Surveys → Fact Sheets (structured) Broadcasts + Discover portal + Survey Approvals
Framework alignment TIME + 6R first-class fields ✅ (Gartner-standard) Configurable — not pre-built
Free-form modeling Free Draw — non-FS shapes are visual-only ⚠️ All drawn relations are queryable graph data
Stakeholder roadmap sharing Unlimited viewer users, dashboards ✅ Presentations + Discover consumer app
🚨
Critical UX Debt: Free Draw shapes are "visual-only" Shapes in LeanIX Free Draw that aren't Fact Sheet containers don't appear in reports or other views. This breaks the designer expectation that "what I draw = what the system knows." Users who sketch a target-state diagram expect it to be queryable data.

5

UX Action Items — Prioritized

  • 1
    Ship a stakeholder-facing ADR consumer view Ardoq's "Decision Records" Viewpoint in Discover is the benchmark. LeanIX needs a zero-training read-only view where non-EA stakeholders can see which decisions affect their applications — without needing to understand Fact Sheets.
  • 2
    Surface Jira traceability on ADR detail pages Ardoq wins on ADR→initiative→Jira→delivery-status traceability. LeanIX should expose this chain visually on the decision detail page: Decision status → linked Initiative → Jira epic status — without leaving LeanIX.
  • 3
    Fix Free Draw "visual-only" expectation gap The current behavior — drawn shapes that aren't Fact Sheets don't appear in reports — breaks the mental model for anyone familiar with Lucidchart or Miro. Either surface a clear in-canvas warning ("this shape won't appear in reports") or make all drawn elements promotable to Fact Sheets in one click.
  • 4
    Make the AI Agent Radar legible to CISOs The EU AI Act risk-tier fields exist in the metamodel extension — the UX gap is a summary view. Design a one-page "AI Compliance Posture" report: N agents by risk tier, N awaiting approval, N violating policy. This is the executive artifact that no competitor ships yet.
  • 5
    Spike: Git-native ADR round-trip Neither LeanIX nor Ardoq syncs with Markdown-in-Git. A proof-of-concept that imports MADR files from a repo into Architecture Decisions (and writes back status changes) would leapfrog the category and win developer-heavy accounts. Structurizr already does this — LeanIX should too.
